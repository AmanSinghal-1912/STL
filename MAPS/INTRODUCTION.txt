MAPS::::
- stored in form of key-value pairs...ek key ka do value nhi ho sakta...
- associative container -> elements sorted way mai store hota hai...
- By default, ascending order of keys...
- #include<map> is required...
- No duplicates...
- Passing maps by value is a nightmare....prefer passing by reference...

SYNTAX TYPE::
map<key_type,val_type,comp> m1; 
key_type, val_type => dtype...
comp => custom comparator to define the order...

COMPLEXITIES::
Inserting	O(log n)
Deleting	O(log n)
Accessing	O(log n)
Finding     O(log n)
Update element by key	O(log n)
Traversing 	O(n)
