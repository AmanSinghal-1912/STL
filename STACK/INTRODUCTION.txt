STACKS::::
- LIFO structure -> Last In First Out...
- Insertion and deletion can be done only from one end...
- Recently inserted element is removed first and first inserted element will be removed last...
- #include<stack> is required...
- Initialization list is not supported by stack...
- Top element can only be accessed...

SYNTAX::
- stack <dtype> name; 

COMPLEXITIES::
- Insert an element (push)  	O(1)
- Delete an element (pop)	    O(1)
- Access top element (peek)  	O(1)
- Traverse the stack	        O(n)
